# テスト品質ルール

## 禁止事項

### 1. テストのスキップ・無効化
- `test.skip()` や `it.skip()` の追加禁止
- `xdescribe` / `xit` への変更禁止
- テストファイルの削除禁止

### 2. アサーションの弱体化
- `expect().toBe()` → `expect().toBeTruthy()` への変更禁止
- アサーションの削除禁止
- `any()` / `anything()` への安易な変更禁止

### 3. Lint/型チェック設定の緩和
- `eslint-disable` の追加禁止（正当な理由なし）
- `@ts-ignore` / `@ts-expect-error` の追加禁止
- 設定ファイルのルール緩和禁止

## 正しい対応

テストが失敗した場合:
1. **実装を修正**してテストを通す
2. **テスト自体が間違っている場合**は、修正理由を明記してから修正
